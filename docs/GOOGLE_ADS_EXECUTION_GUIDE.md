**솔직한 평가:**

## Phase 0: 사전 준비 (광고 집행 전)

### Step 1: Google Ads 계정 생성

1. **Google Ads 계정 등록**
   - URL: https://ads.google.com/
   - Google 계정으로 로그인
   - 비즈니스 정보 입력:
     - 비즈니스명: `Kusuridoki` (또는 `くすりどき`)
     - 타임존: `Asia/Tokyo`
     - 통화: `JPY`
   - 결제 방법: 크레딧카드 등록

2. **"スマートモード" 회피**
   - 초기 설정에서 Google이 "スマートキャンペーン"를 권유함
   - 반드시 **"エキスパートモード"로 전환** (하단 링크)
   - 스마트모드는 설정 자유도가 낮아 App Campaign 제어 불가

### Step 2: Firebase 연동 (전환 추적)

Google Ads App Campaign의 핵심은 **전환 추적**. Firebase Analytics와 연동하여 설치 이후 행동까지 추적한다.

1. **Firebase 프로젝트 연결**
   - Google Ads > 도구 > 데이터 관리자 > 연결된 계정
   - Firebase 선택 > `kusuridoki` 프로젝트 연결
   - 참고: Firebase 프로젝트와 Google Ads 계정이 동일한 Google 계정에 있어야 연결 가능

2. **Firebase Analytics 이벤트 확인**

   자동 추적 이벤트 (설정 불요):

   | 이벤트 | 설명 | 용도 |
   |--------|------|------|
   | `first_open` | 앱 최초 실행 | 설치 전환 |
   | `app_remove` | 앱 삭제 | 이탈 분석 |

   커스텀 이벤트 (앱 코드에서 로깅 필요):

   | 이벤트 | 트리거 시점 | 전환 가치 |
   |--------|-----------|----------|
   | `medication_added` | 약 등록 완료 | 높음 — 핵심 온보딩 단계 |
   | `reminder_set` | 리마인더 설정 | 높음 — 리텐션 핵심 액션 |
   | `medication_taken` | 복약 기록 탭 | 최고 — D7 리텐션 예측 지표 |

3. **Google Ads에서 Firebase 이벤트를 전환으로 import**
   - Google Ads > 도구 > 측정 > 전환
   - "+ 새 전환 액션" > Firebase
   - 위 3개 커스텀 이벤트를 전환으로 등록
   - `first_open`은 App Campaign이 자동 추적하므로 별도 등록 불요

### Step 3: App Store / Play Store 리스팅 확인

App Campaign은 스토어 리스팅의 텍스트/이미지를 자동 활용하므로, 리스팅 품질이 광고 성과에 직결된다.

기존 `APP_STORE_METADATA.md` 기준 확인 항목:

- [ ] 앱 타이틀에 핵심 키워드 포함 (`くすりどき — 一包化対応 服薬管理`)
- [ ] 서브타이틀에 차별점 명시 (`一包化・個別薬まとめて管理`)
- [ ] 스크린샷 6장 업로드 완료 (일본어)
- [ ] 앱 설명문 첫 3줄이 가치 제안 명확
- [ ] 면책 조항 포함: `⚠️ 医療に関するご注意`
- [ ] Play Store 콘텐츠 등급: `전체이용가`
- [ ] 프라이버시 폴리시 URL 등록

## Phase 1: 캠페인 설계 (¥60,000 배분)

### 예산 배분 전략

```
총 예산: ¥60,000
├── 메인 캠페인: ¥48,000 (80%)
│   └── 4주간 ¥1,700/일 (¥1,700 × 28일 = ¥47,600)
└── 크리에이티브 테스트: ¥12,000 (20%)
    └── 2주째부터 투입, ¥850/일 × 14일
```

**왜 80/20 배분인가:**
- App Campaign은 최소 2주 학습 기간 필요 — 메인 캠페인에 충분한 예산 할당
- 크리에이티브 테스트는 학습 완료 후(3주째) 본격 비교 시작
- ¥1,700/일은 tCPI ¥700 기준 약 2.4건/일 — Google 권장 최소 기준(일 10건)에는 미달하지만, 니치 시장에서는 작동 가능

### 캠페인 구조

```

├── [메인] KSD_App_Install_JP
│   ├── 타겟: 일본 전국, 일본어
│   ├── 입찰: tCPI ¥700
│   ├── 일 예산: ¥1,700
│   └── 에셋 그룹: 텍스트 10개 + 이미지 5개 + 동영상 3개
```

## Phase 3: 의약품 광고 규제 준수

### Google Ads 헬스케어 정책

Google Ads는 헬스케어 관련 광고에 대해 별도 정책을 적용한다.

**Kusuridoki의 분류:**

| 항목 | 판단 | 근거 |
|------|------|------|
| 의약품 광고인가? | **아니오** | 앱 자체가 의약품이 아님 |
| 의료기기 광고인가? | **아니오** | SaMD(소프트웨어 의료기기) 아님 |
| 헬스케어 서비스 광고인가? | **그레이존** | 복약 "관리 도구"로 분류 |
| 사전 인증 필요? | **일반적으로 불요** | 의료기기가 아닌 일반 앱 |

> **참고:** Google Ads 헬스케어 정책은 국가별로 다름. 일본에서 복약 리마인더 앱은 일반적으로 사전 인증 불요. 다만 광고 텍스트가 의료 효과를 주장하면 심사 거부될 수 있음.
>
> 정책 원문: https://support.google.com/adspolicy/answer/176031?hl=ja

## Phase 4: 운영 & 최적화 (4주 타임라인)

### Week 1~2: 학습 기간

**절대 규칙: 건드리지 않는다.**

Google App Campaign은 최초 2주간 머신러닝 학습을 진행한다. 이 기간에 설정을 변경하면 학습이 리셋된다.

| 해도 되는 것 | 하면 안 되는 것 |
|-------------|--------------|
| 일 예산 20% 이내 조정 | 입찰 전략 변경 |
| 에셋 추가 (기존 유지하면서) | 기존 에셋 삭제/교체 |
| 성과 데이터 확인 | 타겟팅 변경 |
| Firebase 이벤트 확인 | 캠페인 일시정지 후 재시작 |

**1주차 확인 사항:**
```
□ 광고가 정상 배신되고 있는가 (imp > 0)
□ Firebase 전환 이벤트가 정상 수신되는가
□ 심사 거부된 에셋이 없는가
□ 일 소비가 예산의 ±30% 이내인가
```

### Week 3: 분석 & 크리에이티브 테스트 시작

**분석 항목:**

| 지표 | 확인 방법 | 조치 |
|------|----------|------|
| 에셋별 성과 | Google Ads > 에셋 보고서 | "低" 등급 에셋 교체 후보 |
| CPI | 전환 > 비용/전환 | ¥700 초과 시 tCPI 유지, ¥1,000 초과 시 크리에이티브 점검 |
| CTR | 에셋 그룹 보고서 | 1% 미만 시 헤드라인/이미지 교체 |
| 전환 이벤트 분포 | Firebase > 이벤트 | `medication_added` 전환율 확인 |

**테스트 캠페인 시작:**
- 메인 캠페인에서 "低" 등급 에셋을 신규 에셋으로 교체
- 테스트 캠페인 (¥850/일) 시작 — 메인과 다른 크리에이티브 세트
- 2개 캠페인의 CPI/CTR 비교

### Week 4: 최적화 & 마무리

| 액션 | 조건 |
|------|------|
| 고성과 에셋 유지 | "最良" 등급 에셋 |

## 부록 A: Google Ads App Campaign vs Apple Search Ads vs LINE Ads 비교

| 항목 | Google App Campaign | Apple Search Ads | LINE Ads |
|------|-------------------|-----------------|----------|
| **플레이스먼트** | Search, YouTube, Display, Play Store | App Store 검색 결과 | Talk, NEWS, VOOM, 파트너앱 |
| **타겟팅** | 자동 (ML 기반) | 키워드 (수동/자동) | 연령, 성별, 관심사, 행동 |
| **크리에이티브 제어** | 에셋 제출 → Google이 조합 | Custom Product Page 연동 | 직접 디자인 (이미지+텍스트) |
| **최적화 방식** | 자동 (tCPI/tCPA) | 수동/자동 입찰 | 자동/수동 입찰 |
| **CPI (일본 추정)** | ¥500~1,500 | ¥300~700 (니치 키워드) | ¥200~500 |
| **학습 기간** | 2주 | 1~2주 | 1~2주 |
| **최소 권장 예산** | 일 ¥1,000+ | 일 ¥500+ | 월 ¥10,000+ |
| **강점** | 자동 최적화, 넓은 도달 | 높은 설치 의도, 낮은 CPI | 일본 시니어 도달률 1위 |
| **약점** | 세밀한 타겟팅 불가 | iOS만 가능 | 최소 예산 높음 |
| **Kusuridoki 적합도** | 범용 인스톨 + 크리에이티브 테스트 | 핵심 ASO 키워드 포착 | 케어기버/시니어 집중 공략 |

## 부록 C: 트러블슈팅

### 계정 관련

| 문제 | 원인 | 해결 |
|------|------|------|
| 계정 정지 | 정책 위반 감지 | 이의 신청 폼 제출, 위반 항목 수정 후 재심사 요청 |
| 결제 실패 | 카드 한도 초과 / 만료 | 결제 수단 업데이트, 대안 카드 등록 |
| "スマートモード"에서 전환 불가 | 초기 설정 오류 | 새 계정 생성 후 "エキスパートモード"로 시작 |

### 심사 관련

| 문제 | 원인 | 해결 |
|------|------|------|
| 헬스케어 정책 위반으로 심사 거부 | 광고 텍스트에 의료 효능 암시 | Phase 3의 OK/NG 체크리스트로 표현 수정 |
| "제한적 콘텐츠" 경고 | 건강 관련 광고로 분류 | 면책 조항 명시, "관리 도구"로 포지셔닝 강조 |
| 이미지 텍스트 과다 거부 | 이미지 내 텍스트 비율 초과 | 텍스트를 헤드라인/설명문으로 이동, 이미지는 비주얼 중심 |
| 랜딩 페이지 불일치 | 광고 내용과 스토어 페이지 불일치 | 스토어 설명문과 광고 카피 정합성 확인 |

### 전환 추적 관련

| 문제 | 원인 | 해결 |
|------|------|------|
| 전환 수 0 (인스톨 발생했는데) | Firebase 연동 미완료 | Google Ads > 연결된 계정에서 Firebase 연결 확인 |
| 커스텀 이벤트 미수신 | 앱 코드에서 이벤트 미발송 | Firebase Analytics 디버그 모드로 이벤트 발송 확인 |
| iOS 전환 수 과소 집계 | ATT (App Tracking Transparency) | SKAdNetwork 전환값 설정 확인 — App Campaign은 SKAdNetwork 자동 지원 |

### 성과 관련

| 문제 | 원인 | 해결 |
|------|------|------|
| CPI가 ¥1,500+ | 경쟁 심화 or 크리에이티브 품질 낮음 | 에셋 "低" 등급 교체, tCPI 목표 상향 검토 |
| 배신량 극소 (imp < 1,000/일) | 일 예산 너무 낮거나 tCPI 너무 낮음 | tCPI 10~20% 상향, 또는 에셋 추가 |
| 설치 후 Day 1 이탈률 > 70% | 앱 온보딩 문제 | 앱 내 온보딩 플로우 개선 (광고 문제 아님) |
