# providers — Provider Tests

## Purpose
Unit and integration tests for `lib/data/providers/` — verifies Riverpod provider state transitions, async loading, error handling, and mutation operations.

## For AI Agents

### Working In This Directory
- Use `ProviderContainer` with `overrides` to inject mock services
- Each provider test file has a corresponding `.mocks.dart` generated by mockito
- After adding `@GenerateMocks`: `dart run build_runner build --delete-conflicting-outputs`
- Call `container.dispose()` in `tearDown`
- Run: `flutter test test/data/providers/`

### Common Pattern
```dart
final container = ProviderContainer(
  overrides: [
    medicationRepositoryProvider.overrideWithValue(mockRepository),
  ],
);
addTearDown(container.dispose);

final state = await container.read(medicationListProvider.future);
expect(state, isNotEmpty);
```

### Critical Notes
- `kPremiumEnabled = false` — all premium-related provider tests must be `skip: true`
- Use `setupFirebaseAuthMocks()` from `test/mock_firebase.dart` for auth provider tests
